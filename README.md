# Campus Buzz — Campus Coordination Platform

This repository contains our solution for **BuildSprint**, focused on building scalable, real-world platforms that solve campus coordination and recruitment challenges.
  
## Description

Campus life coordination is fragmented across WhatsApp groups, notice boards, and informal communication. Campus Buzz creates a verified student-first social platform where students can coordinate, clubs can communicate, events can be discovered, and complaints can be raised.  
<br>  
### The platform supports three roles:
#### 1. Student  
- Create Campus Buzz posts  
- Join coordination rooms   
- Raise complaints  
- View events    

#### 2. Club / Committee  
- Publish official announcements  
- Create events  
- Embed forms in posts  

#### 3. Admin  
- Manage users  
- Moderate content  
- Resolve complaints  
- Create events  

Role-based permissions are enforced at the backend level.  
  
<hr width="50%">  
  
### Campus Buzz Feed  

Students can create posts containing:  
&nbsp; - Image  
&nbsp; - Title  
&nbsp; - Description  
&nbsp; - Mandatory hashtag  

Supported hashtags:

| Hashtag | Action |
|---------|--------|
| #foodsplit | Creates live food coordination room |
| #cabsplit | Creates cab sharing room |
| #resell | Creates buyer-seller discussion room |
| #lost / #found | Displays contact information |  
  
<hr width="50%">
  
### Auto Expiry System

For temporary posts:  

&nbsp; - Default expiry: 24 hours  
&nbsp; - Custom expiry: 10 minutes to 2 days  
&nbsp; - Remaining timer displayed on posts  
&nbsp; - Background jobs handle automatic deletion    
  
<hr width="50%">  
   
### Event Calendar   

Campus-wide calendar containing:    
  
&nbsp; - Event name    
&nbsp; - Date    
&nbsp; - Time    
&nbsp; - Venue    
&nbsp; - Description    

Features:   
  
- Clubs can create and manage events    
- Students can request event creation    
- Events can be linked with official posts      
  
<hr width="50%">  
  
### Complaint System   

*Students can raise anonymous complaints.*
  
Features:  
&nbsp; - Public anonymous view  
&nbsp; - Backend stores actual user identity  
&nbsp; - User can mark complaints resolved  
&nbsp; - Admin can moderate complaints
